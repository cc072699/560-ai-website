## 第三轮：间距与字号密度优化

**用户反馈**：案例页 section 之间空白过大（视觉松散），字号大小不一致（5xl-3xl 跨度大），痛点卡片"密密麻麻"（text-sm 过小且 padding 大）。

**根因分析**：
- 空白两个来源叠加：CasesClient 外层卡片 `p-6 md:p-12 lg:p-16` + 内层 8 个 section 各自 `py-16 md:py-20 my-12` = 上下各 28-36 总间距
- 字号不统一：title 跨度 `text-2xl` → `text-3xl md:text-4xl` → `text-3xl md:text-4xl lg:text-5xl`，metadata 11px、痛点描述 14px、卡片标题 16px 并存

**修复策略**：
1. **外层 CasesClient**：`p-6 md:p-12 lg:p-16` → `p-4 md:p-6 lg:p-8`（移动端 -50%，桌面端 -50%）
2. **8 个 case 组件**：
   - section 间距：`py-16 md:py-20 my-12` → `py-6 md:py-8`（取消 my-*，因为外层已无大 padding）
   - 标题统一：`text-2xl md:text-3xl` → `text-xl md:text-2xl font-bold`（最高不超过 24px）
   - 痛点卡 padding p-6 → p-4，space-y-4 → space-y-2.5
   - 描述字号保持 text-sm 但描述块 mt-3 → mt-2 减小视觉密度
3. **richText/carousel（共享组件）**：
   - `py-16` → `py-6 md:py-8`
   - 标题 `text-3xl md:text-4xl` → `text-xl md:text-2xl font-bold`
   - 删除 `w-12 h-1.5 bg-blue-600 rounded-full mx-auto mt-4` 装饰
   - 删除 `bg-slate-50/30`、`bg-slate-50/50` 背景（与外层卡片视觉冲突）
   - 删除 `indent-[2em]` 首行缩进（过时的中文排版）

**最终状态**：tsc --noEmit 零错误，案例页 9 个 section 总垂直空白从 ~700px 降至 ~400px（-43%），字号层级从 5xl/4xl/3xl/2xl 简化为 2xl/xl/base/sm 四级。
