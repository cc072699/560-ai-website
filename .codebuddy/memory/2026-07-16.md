## 🟡 网格排版"上多下少"问题修复

**问题**：3 项 carousel 显示成 2+1，4 项 challenges 显示成 3+1，5 项 metrics 显示成 3+2——典型的固定列数与项目数不匹配造成的"孤项"现象。

**根因**：硬编码 `md:grid-cols-3` 这种固定列数在 3/4/5 项时必然造成不均匀分布。

**修复方案**：使用 CSS Grid `auto-fit` + `minmax` 自适应：
```tsx
<div
  className="grid gap-8 max-w-6xl mx-auto"
  style={{ gridTemplateColumns: 'repeat(auto-fit, minmax(340px, 1fr))' }}
>
```

**关键参数**：
- `minmax(340px, 1fr)` — 每列最小 340px，最大等分；超过 340px 一倍就换列
- `minmax(220px, 1fr)` — 用于更紧凑的指标卡片
- 配合 `max-w-6xl` 容器宽度

**应用位置**（DetailSectionsRenderer.tsx）：
- `DetailCarousel` (line 691): `minmax(340px, 1fr)` + `max-w-6xl`
- `DetailCaseMetrics` (line 1512): `minmax(220px, 1fr)` + `max-w-6xl`
- `DetailCaseChallenges` (line 1389): 直接 `md:grid-cols-2`（4 项固定 2x2）

**3 项效果**：宽屏自动 3 列 1 行，窄屏 2 列（2+1 不可避免但出现频率低）
**4 项效果**：中宽屏 2x2 整齐排列
**5 项效果**：宽屏 5 列 1 行
