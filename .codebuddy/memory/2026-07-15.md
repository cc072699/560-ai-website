## 修复：管理后台多图片上传组件文件错位 Bug

**问题**：产品管理后台中，同一页面有多个 ImageUpload/VideoUpload 组件时，在第二个、第三个等组件中上传文件，文件会被错误地传到第一个组件中。

**根因**：`ImageUpload.tsx` 和 `VideoUpload.tsx` 使用了硬编码的 DOM `id`（`image-file-input` / `video-file-input`）。当页面有多个实例时，`document.getElementById('image-file-input')` 总是找到 DOM 中第一个匹配的元素，导致所有实例的 click 事件都触发了同一个隐藏的 file input。

**修复**：将 `document.getElementById()` 改为 `useRef()`，每个组件实例持有自己独立的 ref：
- `ImageUpload.tsx`：添加 `const fileInputRef = useRef<HTMLInputElement>(null)`，用 `fileInputRef.current?.click()` 和 `ref={fileInputRef}` 替代原来的 `id` + `getElementById` 模式
- `VideoUpload.tsx`：同样修复
