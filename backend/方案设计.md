## 一、方案总览

### 核心选型：Next.js 全栈 + JSON 文件数据源 + 独立管理后台

```
┌─────────────────────────────────────────────────────────────────┐
│                        架构总览                                  │
│                                                                  │
│  ┌──────────────┐     ┌──────────────────────┐                  │

│  │  (现有)      │     │  密码保护 / 本地账号   │                  │
│  └──────┬───────┘     └──────────┬───────────┘                  │
│         │                        │ 读写                          │
│         │ 读取（SSR/ISR）         ▼                              │
│         │              ┌──────────────────────┐                  │
│         └─────────────▶│   数据层（JSON文件）  │                  │

│                        └──────────────────────┘                  │

│                         │   媒体文件存储       │                  │

## 二、为什么选择这套方案

| 评估维度 | JSON文件 + Next.js全栈 | 说明 |
|---------|------------------------|------|
| **维护难度** | ⭐⭐⭐⭐⭐ 极简 | 不需要懂数据库，Admin界面可视化操作 |
| **部署难度** | ⭐⭐⭐⭐⭐ 极简 | 一台服务器、单一Node进程，无额外服务 |
| **成本** | ⭐⭐⭐⭐⭐ 最低 | 零额外服务费用，无数据库License |
| **安全性** | ⭐⭐⭐⭐ 高 | 管理后台受密码保护，数据不对外暴露 |
| **扩展性** | ⭐⭐⭐ 中 | 内容量大时可无缝迁移至 SQLite/PostgreSQL |
| **适用场景** | 官网 / 展示型网站 | ✅ 与本项目完全匹配 |

### 与其他方案对比

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| WordPress / CMS | 功能全 | 重量级、安全漏洞多、需PHP | ❌ 已排除 |
| Headless CMS (Strapi/Sanity) | 专业CMS体验 | 需单独部署服务、维护复杂 | ❌ 过度设计 |
| 数据库 (MySQL/PostgreSQL) | 企业级 | 需DBA维护、部署复杂 | ❌ 小内容量过度设计 |
| **JSON文件 + Next.js Admin** | 轻量、直接 | 不适合海量数据 | ✅ **最优** |
| Notion/飞书作为数据源 | 易用 | 依赖第三方服务、接口限流 | ⚠️ 备选 |

## 三、技术栈

```
前端（现有）：  Next.js 16 + React 19 + TypeScript + Tailwind CSS

## 五、后台管理界面功能

### 5.1 访问路径与认证

```
https://your-domain.com/admin           → 登录页
https://your-domain.com/admin/dashboard → 后台首页
https://your-domain.com/admin/products  → 产品管理
https://your-domain.com/admin/cases     → 客户案例管理
https://your-domain.com/admin/settings  → 全局配置
```

### 5.2 各管理页面功能

| 页面 | 功能 |
|------|------|
| **仪表盘** | 快速概览，各模块数据条目数量 |
| **产品管理** | 新增/编辑/删除/排序/显示隐藏 产品卡片 |
| **客户案例** | 新增/编辑/删除/排序/显示隐藏 案例，支持图片上传 |
| **全局配置** | 修改公司名称、联系电话、邮箱、地址、版权信息 |
| **图片上传** | 本地图片文件上传管理（支持 JPG/PNG/WebP） |

### 5.3 界面设计原则

- **无需技术背景**：所见即所得的表单操作，无需接触代码
- **操作保护**：删除操作需二次确认
- **变更即时生效**：修改后前端页面在下一次请求时自动更新（ISR，30秒内）
- **历史回溯**：JSON文件纳入 Git 管理，可追踪每次改动

## 六、技术实现方案

### 6.1 API Routes 设计

```
POST   /api/admin/login              → 管理员登录，签发 JWT Cookie
POST   /api/admin/logout             → 登出
GET    /api/admin/products           → 获取所有产品
POST   /api/admin/products           → 新增产品
PUT    /api/admin/products/[id]      → 更新产品
DELETE /api/admin/products/[id]      → 删除产品
GET    /api/admin/cases              → 获取所有案例
POST   /api/admin/cases              → 新增案例
PUT    /api/admin/cases/[id]         → 更新案例
DELETE /api/admin/cases/[id]         → 删除案例
GET    /api/admin/site               → 获取全局配置
PUT    /api/admin/site               → 更新全局配置
POST   /api/admin/upload             → 图片上传
```

### 6.2 认证方案

```
工具：NextAuth.js v5（App Router 兼容）
方式：Credentials Provider（用户名+密码）
存储：账号信息存入 .env.local（1-2个管理员账号）
      密码使用 bcrypt 哈希存储
会话：JWT Cookie，httpOnly，7天有效期
```

> **为何不用数据库存账号**：官网维护人员通常只有 1-2 人，
> 账号直接存入环境变量更安全、更简单，无需管理员账号的增删改查界面。

### 6.3 前端数据获取方式

```typescript
// /src/lib/data.ts - 数据读取层
import fs from 'fs/promises'
import path from 'path'

export async function getProducts() {
  const filePath = path.join(process.cwd(), 'data', 'products.json')
  const raw = await fs.readFile(filePath, 'utf-8')
  return JSON.parse(raw)
}

export async function getCases() {
  const filePath = path.join(process.cwd(), 'data', 'cases.json')

}

export async function getSiteConfig() {
  const filePath = path.join(process.cwd(), 'data', 'site.json')
  const raw = await fs.readFile(filePath, 'utf-8')
  return JSON.parse(raw)
}
```

### 6.4 数据写入（API Route 示例）

```typescript
// /src/app/api/admin/products/route.ts
import { writeFile, readFile } from 'fs/promises'
import path from 'path'
import { revalidatePath } from 'next/cache'
import { withAdminAuth } from '@/lib/auth'

// 新增产品
export const POST = withAdminAuth(async (req: Request) => {
  const body = await req.json()
  const filePath = path.join(process.cwd(), 'data', 'products.json')
  const current = JSON.parse(await readFile(filePath, 'utf-8'))
  const newItem  = { ...body, id: Date.now().toString() }
  await writeFile(filePath, JSON.stringify([...current, newItem], null, 2), 'utf-8')
  revalidatePath('/')  // 触发前端缓存刷新
  return Response.json({ success: true, item: newItem })
})
```

## 八、部署方案

### 8.1 单服务器部署（推荐）

```bash
# 服务器环境
操作系统：Ubuntu 22.04 LTS
Node.js：v20 LTS
进程管理：PM2
反向代理：Nginx
SSL：Let's Encrypt（certbot）

# 部署流程
git pull origin main
npm install
npm run build
pm2 restart 560-ai-website
```

### 8.2 Nginx 配置

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 可选：限制 /admin 仅允许内网IP访问
    # location /admin {
    #     allow 192.168.1.0/24;
    #     deny all;
    #     proxy_pass http://localhost:3000;
    # }

    client_max_body_size 10M;
}
```

### 8.3 环境变量配置 `.env.local`

```bash
# 管理员账号（维护人员直接修改此文件）
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=$2b$10$...   # bcrypt哈希值，不存明文

# NextAuth 配置
NEXTAUTH_SECRET=your-random-32-char-secret
NEXTAUTH_URL=https://your-domain.com

# 可选：媒体文件存储（使用云存储时填写）
# OSS_BUCKET=your-bucket
# OSS_REGION=cn-nanjing
# OSS_ACCESS_KEY_ID=...
# OSS_ACCESS_KEY_SECRET=...
```

## 十、维护人员操作手册（简明版）

### 日常更新流程

1. **访问管理后台**  
   浏览器打开 `https://your-domain.com/admin`，输入账号密码

2. **修改产品信息**  
   侧边栏 → 产品介绍 → 点击"编辑" → 修改文字/图片 → 点击"保存"  
   **约30秒后**，官网前台自动更新

3. **新增客户案例**  
   侧边栏 → 客户案例 → 点击"新增案例" → 填写表单 → 上传图片 → 保存

4. **修改联系信息**  
   侧边栏 → 全局设置 → 直接修改 → 保存

### 常见问题

| 问题 | 解决方案 |
|------|---------|
| 修改后前台没有及时更新 | 等待最多30秒，或清除浏览器缓存后刷新 |
| 忘记管理员密码 | 联系技术人员修改服务器上的 `.env.local` 文件 |
| 图片上传失败 | 检查图片大小是否超过 10MB；仅支持 JPG/PNG/WebP 格式 |

## 十一、后续扩展路径

| 当前方案 | 升级方案 | 触发条件 |
|---------|---------|---------|
| JSON文件存储 | SQLite（zero-migration，仍单文件） | 条目 > 500条 |

| 单账号密码 | 多用户权限管理 | 维护团队 > 3人 |
